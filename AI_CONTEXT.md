# 프로젝트 컨텍스트: "갚을까? 저축할까?" (대출 중도상환 vs 정기예금 수익 비교기)

## 1. 프로젝트 개요
- **목적**: 노아에이티에스(주) 채용 사전과제 - 주제 1 (대출 & 관리)
- **핵심 목표**: 여유 자금이 생겼을 때 '대출 중도 상환'과 '정기예금 가입' 중 어떤 것이 경제적으로 이득인지 시뮬레이션하여 의사결정을 지원함.
- **부가 목표**: 대환대출을 고려할 때 대출을 갈아타는 것과 유지하는 것 어떤 것이 경제적으로 이득인지 의사결정 지원.


## 2. 기술 스택 (Tech Stack)
- **Framework**: Next.js 14+
- **Language**: TypeScript
- **Styling**: Tailwind CSS, Shadcn UI (추천)
- **Deployment**: Vercel (CI/CD 연동)

## 3. 핵심 도메인 용어 및 규칙
- **Loan (대출)**: 사용자가 현재 갚아야 할 빚.
- **Prepayment Fee (중도상환수수료)**: 만기 전 대출을 갚을 때 발생하는 금액.
- **Deposit (정기예금)**: 여유 자금을 예치하여 이자를 받는 상품.
- **Interest Tax (이자소득세)**: 예금 이자 수익의 15.4% (반드시 차감 후 세후 수익으로 비교해야 함).

## 4. 코딩 규칙 (Coding Rules)
- **컴포넌트 분리**: UI 컴포넌트는 `components/`에, 비즈니스 로직은 `lib`에 분리하여 작성할 것.
- **정확성 우선**: 금융 계산이므로 모든 금액은 `Math.floor` 등을 사용하여 원 단위 혹은 소수점 처리를 명확히 할 것.
- **에러 핸들링**: 사용자가 음수 금액이나 유효하지 않은 금리를 입력할 경우에 대한 예외 처리를 포함할 것.
